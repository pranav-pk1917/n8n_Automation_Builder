## What this workflow does (plain English)

Stakeholders often want to see keyword priorities in a **spreadsheet**, not only in Supabase. Every night this workflow:

1. Loads each **active client**.
2. Fetches their **scored keyword clusters** (sorted by priority).
3. Fetches recent **API cost** log entries.
4. **Appends rows** to a Google Sheet tab named `Clusters`.
5. Sends a **Slack summary** (cluster count, total API cost, top 3 clusters).

This is a reporting/export step — it does not change scoring logic.

---

## Current features (overview)

| Feature | What it does |
|---------|----------------|
| **Nightly schedule** | Runs at **03:00 UTC** daily (after WF-00 at 02:00). |
| **Active client fetch** | Same filter as WF-00: `onboarding_status = active`. |
| **Scored cluster export** | Clusters with non-null `priority_score`, top 500 by priority. |
| **API cost rollup** | Last 200 cost log rows per client for summary math. |
| **Sheet row builder** | Flattens cluster fields + sync timestamp for Sheets. |
| **Google Sheets append** | Appends to tab `Clusters` in `GOOGLE_SHEETS_ID`. |
| **Slack digest** | Posts client name, cluster count, cost, top clusters. |

**Note:** The live workflow writes to a single **`Clusters`** tab. The export JSON in `exports/wf-05-sheet-sync.json` may describe per-pillar tabs — use the n8n editor as source of truth.

## Data flow (start to end)

```
Nightly_Schedule (cron)
        │
        ▼
┌─────────────────────┐
│ Fetch_Active_       │  GET clients (active)
│ Clients             │  → typically one item per client per run*
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│ Fetch_Scored_       │  GET keyword_clusters for client.id
│ Clusters            │  priority_score NOT NULL, ordered DESC
└─────────┬───────────┘
          ├────────────────────────┐
          ▼                        ▼
┌─────────────────────┐  ┌─────────────────────┐
│ Fetch_API_Costs     │  │ Build_Sheet_Rows    │  One row per cluster
│ (parallel)          │  │                     │
└─────────┬───────────┘  └─────────┬───────────┘
          │                        ▼
          │              ┌─────────────────────┐
          │              │ Write_To_Google_    │  Append → Sheet "Clusters"
          │              │ Sheets              │
          │              └─────────┬───────────┘
          │                        ▼
          │              ┌─────────────────────┐
          └─────────────►│ Build_Summary       │  cluster_count, total_cost, top 3
                           └─────────┬───────────┘
                                     ▼
                           ┌─────────────────────┐
                           │ Send_Slack_Summary  │
                           └─────────────────────┘
```

\*If multiple active clients exist, n8n may run one branch per client item — confirm behavior in Executions when testing multi-client setups.

## Testing tips

1. Run **WF-04** first so clusters have `priority_score` set.
2. Set `GOOGLE_SHEETS_ID` in n8n variables.
3. Share the spreadsheet with the Google OAuth account used in n8n.
4. Execute WF-05 manually once.
5. Confirm new rows in the `Clusters` tab and Slack message in `#seo-tools-hitl`.

---

## Typical issues

| Symptom | Likely cause |
|---------|----------------|
| Sheets node error | OAuth not connected or wrong `GOOGLE_SHEETS_ID` |
| Empty append | No clusters with `priority_score` yet |
| Slack fails | `Slack_SEOTools` not bound on Send_Slack_Summary |
