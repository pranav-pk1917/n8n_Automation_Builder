---

## What this workflow does (plain English)

Every night, this workflow keeps a **catalog of every page** on each active client’s website. For each page it stores:

- URL, title, H1, meta description, and a snippet of visible text
- A **vector embedding** (numeric representation of the page content for similarity search later)

That catalog lives in Supabase table `pages`. Other parts of the system (e.g. cannibalization checks, content matching) can compare keywords against real site pages.

---

## Current features (overview)

| Feature | What it does |
|---------|----------------|
| **Daily schedule** | Runs automatically at **02:00 UTC** every day. |
| **Multi-client loop** | Processes every client with `onboarding_status = active`. |
| **Sitemap crawl** | Reads `/sitemap.xml` per client (up to 200 URLs). |
| **HTML fetch** | Downloads each page (10 URLs per batch). |
| **Meta extraction** | Parses title, H1, meta description, visible text from HTML. |
| **Embeddings** | Calls OpenRouter embeddings API (`text-embedding-3-small`). |
| **Upsert to Supabase** | Writes/updates `pages` (merge on duplicate URL per client). |

**Note:** The version deployed in n8n uses **sitemap URLs only**. The richer export JSON in `exports/wf-00-page-inventory.json` also includes WPGraphQL and pipeline run tracking — those are not in the current live workflow.

## Data flow (start to end)

```
Daily_Schedule (cron)
        │
        ▼
┌─────────────────────┐
│ Fetch_Active_       │  GET clients where onboarding_status = active
│ Clients             │  → one item per client
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ Client_Loop         │  Process one client at a time (batch size 1)
│                     │
│   Build_Client_     │  Build base_url from canonical_domain
│   Context           │  → { client_id, client_name, base_url, config }
│         │
│         ▼
│   Fetch_Sitemap     │  GET base_url/sitemap.xml
│         │
│         ▼
│   Parse_Sitemap_    │  Parse <loc> tags → up to 200 items with page_url
│   URLs              │
│         │
│         ▼
│   Page_Loop         │  Batches of 10 pages
│     ├─ Crawl_Page   │  GET page HTML
│     ├─ Extract_     │  title, h1, meta_description, visible_text
│     │  Page_Meta    │
│     ├─ Embed_Page   │  OpenRouter embeddings
│     └─ Upsert_Page  │  POST → Supabase pages (merge duplicates)
│     └─ (next batch) │
│                     │
│   (next client)     │
└─────────────────────┘
```

## Node reference

| Node | Type | What it does |
|------|------|--------------|
| **Sticky Note** | Sticky | Canvas documentation. |
| **Daily_Schedule** | Schedule Trigger | Fires daily at 02:00 UTC. |
| **Fetch_Active_Clients** | HTTP Request | Lists active clients from Supabase. |
| **Client_Loop** | Split in Batches | One client per iteration. |
| **Build_Client_Context** | Code | Turns `canonical_domain` into `https://...` base URL. |
| **Fetch_Sitemap** | HTTP Request | Downloads sitemap XML (15s timeout). |
| **Parse_Sitemap_URLs** | Code | Extracts HTTP URLs from sitemap (max 200). |
| **Page_Loop** | Split in Batches | 10 pages per batch. |
| **Crawl_Page** | HTTP Request | GET `page_url` HTML (10s timeout). |
| **Extract_Page_Meta** | Code | Regex extraction from HTML. |
| **Embed_Page** | HTTP Request | OpenRouter `/v1/embeddings` for page text. |
| **Upsert_Page** | HTTP Request | POST to `pages` with `Prefer: resolution=merge-duplicates`. |

## What each page record contains (Supabase `pages`)

| Field | Source |
|-------|--------|
| `client_id` | From client loop |
| `url` | Sitemap URL |
| `title` | `<title>` tag |
| `h1` | First `<h1>` |
| `meta_description` | meta description tag |
| `embedding` | Vector from OpenRouter |
| `last_crawled_at` | Timestamp when synced |

## Testing tips

1. Ensure at least one client has `onboarding_status = active` in Supabase.
2. Bind credentials on all HTTP nodes.
3. Click **Execute workflow** (manual run) — watch **Executions**.
4. Confirm new/updated rows in Supabase `pages` for that client.
5. If sitemap fails, check the client’s `canonical_domain` and that `/sitemap.xml` is reachable.

---

## Typical issues

| Symptom | Likely cause |
|---------|----------------|
| No pages written | Client not `active`, or sitemap empty/unreachable |
| Embed fails | OpenRouter credential or quota |
| Timeout on crawl | Slow site; increase timeout in Crawl_Page if needed |
