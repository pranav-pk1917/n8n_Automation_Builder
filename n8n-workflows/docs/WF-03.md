## What this workflow does (plain English)

After filtering, you have a list of **good keywords**. This workflow:

1. Groups similar keywords into **clusters** using a Python service (HDBSCAN on embeddings).
2. Uses AI to label each cluster on **three axes**:
   - **Service pillar** (what you sell)
   - **Vertical niche** (industry)
   - **Content theme** (topic angle)
3. Saves clusters to Supabase.
4. Alerts Slack if the AI suggests a **new pillar or niche** not in your taxonomy.

---

## Current features (overview)

| Feature | What it does |
|---------|----------------|
| **Passed-keyword fetch** | Loads classifications with `decision = passed` (joins keyword text, volume, KD). |
| **Cluster payload build** | Packages keywords for the Python worker. |
| **Python /cluster** | Railway worker runs HDBSCAN clustering. |
| **Per-cluster LLM labeling** | Gemini assigns 3-axis taxonomy + human-readable label. |
| **Cluster persistence** | Writes rows to `keyword_clusters`. |
| **Taxonomy suggestions** | If LLM sets `suggest_new_pillar` or `suggest_new_niche`, posts Slack alert. |
| **Webhook response** | Returns `{ ok: true }`. |

## Node reference

| Node | Type | What it does |
|------|------|--------------|
| **Sticky Note** | Sticky | Canvas documentation. |
| **Cluster_Webhook** | Webhook | Starts clustering. |
| **Validate_Input** | Code | Requires `client_id`. |
| **Fetch_Passed_Keywords** | HTTP Request | Supabase join query (limit 1000). |
| **Build_Cluster_Payload** | Code | Builds worker request body. |
| **Call_Python_Cluster** | HTTP Request | POST to `/cluster` with auth header. |
| **Parse_Clusters** | Code | Splits worker response into per-cluster items. |
| **LLM_Label_Cluster** | HTTP Request | OpenRouter JSON labeling per cluster. |
| **Parse_LLM_Label** | Code | Parses LLM; adds cost estimate. |
| **Write_Cluster** | HTTP Request | INSERT `keyword_clusters`. |
| **IF_Taxonomy_Suggestion** | If | `suggest_new_pillar` OR `suggest_new_niche`. |
| **Send_Taxonomy_Suggestion** | HTTP Request | Slack alert to review taxonomy. |
| **Respond_Cluster_OK** | Respond to Webhook | `{ ok: true }`. |
