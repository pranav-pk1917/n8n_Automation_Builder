## What this workflow does (plain English)

Each **keyword cluster** needs a business score: how well does it match the clientвҖҷs ideal customer (ICP), and how commercially valuable is it?

This workflow:

1. Loads clusters that have **not been scored yet** (`icp_score` is null).
2. Checks a simple **cost ceiling** so a run does not blow the API budget.
3. Fetches **Google SERP** results via SerpAPI for each clusterвҖҷs label.
4. Uses AI to score **ICP fit** and **commercial intent** (0вҖ“1 scale).
5. Computes a **priority_score** (weighted formula) for ranking content work.
6. Updates the cluster in Supabase and logs LLM cost.

---

## Current features (overview)

| Feature | What it does |
|---------|----------------|
| **Unscored cluster fetch** | Up to 100 clusters per run with `icp_score IS NULL`. |
| **Cost ceiling guard** | Estimates ~$0.005/cluster; throws if estimate exceeds $50 budget. |
| **SerpAPI lookup** | Top 5 organic results for cluster label (US/en). |
| **LLM ICP scoring** | Gemini returns `icp_score`, `commercial_intent_score`, reasoning. |
| **Priority score formula** | Weighted blend of ICP, commercial intent, keyword count proxy, KD placeholder. |
| **Cluster update** | PATCH `keyword_clusters` with scores + `scored_at`. |
| **API cost logging** | INSERT into `api_cost_log` for WF-04 LLM usage. |
| **Webhook response** | Returns `{ ok: true }`. |

---

## Priority score formula (deployed version)

```
priority_score =
  (icp_score Г— 0.35) +
  (commercial_intent_score Г— 0.25) +
  (volume_norm Г— 0.20) +
  (0.20)   вҶҗ fixed KD component placeholder in current code
```

Where `volume_norm = min(keyword_count / 100, 1.0)`.

Higher `priority_score` = work on this cluster sooner.
