**Open in n8n:** https://n8n-webley-u35816.vm.elestio.app/workflow/3POshJ2qtKokamIZ

---

## What this workflow does (plain English)

You export keywords from a tool like Ahrefs or Semrush as a **CSV file**. This workflow:

1. Accepts the CSV text plus a `client_id`.
2. Parses and normalizes each row (lowercase keyword, optional volume/KD).
3. **Skips duplicates** — keywords already in the database or in the “graveyard” (permanently rejected list).
4. Inserts only **new** keywords into `raw_keywords` with status `raw`.

Those raw keywords are the input for **WF-02** (filtering).

---

## Current features (overview)

| Feature | What it does |
|---------|----------------|
| **Webhook ingest** | On-demand trigger with CSV in the request body. |
| **CSV validation** | Requires header row + `keyword` column; optional `volume`, `kd`. |
| **Keyword normalization** | Trims and lowercases keywords. |
| **Existing keyword dedupe** | Loads all `raw_keywords` for the client. |
| **Graveyard dedupe** | Loads `keyword_graveyard` so dead keywords never return. |
| **Bulk insert** | One Supabase POST with all new rows. |
| **Webhook response** | Returns `{ "ok": true }` when done. |

## Input

```json
{
  "client_id": "uuid-from-supabase-clients-table",
  "csv_content": "keyword,volume,kd\ndigital marketing agency,1000,45\nseo services,500,38"
}
```

| Field | Required | Notes |
|-------|----------|-------|
| `client_id` | Yes | UUID of the client in `clients` table. |
| `csv_content` | Yes | Full CSV as a single string (include header row). |

**CSV rules:**

- First row must be headers.
- Must include a column named `keyword` (case-insensitive after parse).
- `volume` and `kd` columns are optional.

## Data flow (start to end)

```
POST wf-01-ingest
        │
        ▼
┌─────────────────────────┐
│ Validate_And_Parse_CSV  │  Parse CSV → many items (one per keyword row)
└─────────┬───────────────┘
          │  each item: { client_id, keyword, volume, kd }
          ├──────────────────────────┐
          ▼                          ▼
┌─────────────────────┐    ┌─────────────────────┐
│ Fetch_Existing_     │    │ Fetch_Graveyard     │  (parallel branches)
│ Keywords            │    │                     │
└─────────┬───────────┘    └─────────┬───────────┘
          │                          │
          └──────────┬───────────────┘
                     ▼
┌─────────────────────────┐
│ Dedupe_And_Prepare_     │  Keep only keywords NOT in existing or graveyard
│ Insert                  │  Add status: raw, ingested_at
└─────────┬───────────────┘
          │  0 items = nothing to insert (still OK)
          ▼
┌─────────────────────────┐
│ Bulk_Insert_Raw_        │  POST array → raw_keywords
│ Keywords                │
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│ Respond_Insert_OK       │  { "ok": true }
└─────────────────────────┘
```

## Node reference

| Node | Type | What it does |
|------|------|--------------|
| **Sticky Note** | Sticky | Canvas documentation. |
| **Ingest_Webhook** | Webhook | Receives POST with `client_id` + `csv_content`. |
| **Validate_And_Parse_CSV** | Code | Validates input; outputs one n8n item per keyword. |
| **Fetch_Existing_Keywords** | HTTP Request | GET all keywords for client (`executeOnce`). |
| **Fetch_Graveyard** | HTTP Request | GET graveyard keywords for client (`executeOnce`). |
| **Dedupe_And_Prepare_Insert** | Code | Filters to new-only rows with `status: raw`. |
| **Bulk_Insert_Raw_Keywords** | HTTP Request | POST all new rows in one request (`executeOnce`). |
| **Respond_Insert_OK** | Respond to Webhook | JSON success response. |
